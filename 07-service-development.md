# Разработка сервисов

На Саммите вы спроектировали API. Теперь пишем код.

Задание: берёте модуль, открываете спеку, делаете сервис, который ей соответствует. Все эндпоинты, не часть. Если чего-то в спеке нет — проектируете сами и оформляете PR в [neomarket-protocols](https://github.com/URFU2026-NeoMarket/neomarket-protocols). Без PR в протокол реализация не засчитывается.

---

## Какой модуль взять

| Модуль | Эндпоинтов | Rep за M1* |
|--------|-----------|------------|
| **B2B Seller Cabinet** | 7 | ~840 |
| **Moderation** | 4 | ~640 |
| **B2C Buyer** | 27+ | ~1120 |

*\*Rep ориентировочный, может быть пересмотрен.*

Команда берёт от 1 до 3 модулей. Больше модулей — больше Rep.

**B2B — логичный старт.** Кабинет продавца: товары, SKU, категории. Самостоятельный модуль, ни от кого не зависит.

**Moderation** — админка техподдержки, проверяет товары из B2B. Эндпоинтов мало, но без работающего B2B модерировать нечего. Берите вторым модулем, не первым.

**B2C** — каталог + корзина + заказы. Самый большой объём, тоже зависит от B2B (читает товары оттуда). Решайте исходя из сил команды, а не амбиций.

## Что сделать

1. Выбрать модуль (команда решает вместе)
2. Продумать схему БД — какие таблицы, как связаны. Миграции в репозиторий
3. Написать все эндпоинты из спеки. Не заглушки — с базой, с валидацией, с правильными HTTP-кодами
4. Завернуть в Docker

### Стек — на ваш выбор

Django, FastAPI, Spring Boot, Express, NestJS — что знаете, на том и пишите. БД — PostgreSQL (на нём будет прод).

### Пример структуры

Рекомендуем src-based layout — весь код приложения в `src/`, конфиг и инфраструктура в корне:

```
my-service/
├── src/                  # код сервиса
│   ├── models/           # модели данных / ORM
│   ├── routes/           # обработчики эндпоинтов
│   ├── services/         # бизнес-логика
│   └── ...
├── migrations/           # миграции БД
├── Dockerfile
├── docker-compose.yml    # сервис + PostgreSQL
└── README.md             # модуль, стек, как запустить
```

### Проверка готовности

- `docker compose up` — поднимается сервис и база
- эндпоинты отвечают по спеке (структура JSON, коды ответов)
- README — какой модуль, как запустить

## С чего начать (прямо сейчас)

1. Откройте Swagger UI, выберите модуль, пролистайте
2. Создайте репо, накидайте `docker-compose.yml` с вашим фреймворком + PostgreSQL
3. Сделайте один GET-эндпоинт — пусть даже с захардкоженным ответом. Главное — всё запускается

Дальше по одному эндпоинту. Сначала GET-ы, потом POST/PUT.

## FAQ

**Можно поменять модуль?**
Да. Rep за результат, не за намерение.

**Как делить работу?**
Captain решает. Можно по эндпоинтам, можно по слоям (БД / логика / API).

**Тесты нужны?**
Для M1 не обязательно. На следующих этапах автотесты по контрактам будут использоваться для приёмки.
